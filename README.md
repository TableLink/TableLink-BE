## 프로젝트 소개

### 프로젝트 기간 및 인원

- 기간: 2024.10.06 - 2024.11.
- 인원: 1명(개인 프로젝트)

### 프로젝트 개요

- **프로젝트 이름**: TableLink
- **프로젝트 초기 목표**: JWT 방식의 인증 인가 로직을 구축하고 음식점을 예약할 수 있는 서비스를 개발 및 음식점에 대한 리뷰를 남기고 자유롭게 대화할 수 있는 커뮤니티
  플렛폼 개발
- **축소된 프로젝트 목표**: JWT 방식의 인증 인가 로직을 구축하고 Kakao Map API를 활용하여 음식점에 대한 리뷰를 남기고 자유롭게 대화를 나누며 나만의 맛집
  지도를 공유할 수 있는 커뮤니티 사이트 개발

## TableLink 프로젝트 요구사항 명세

### 1. 회원 관리 및 인증
- **회원 가입**: 사용자는 이메일, 비밀번호, 닉네임 등 기본 정보를 입력하여 가입.
- **회원 로그인**: 로그인 시 이메일/비밀번호를 통해 인증을 받고, 인증된 사용자는 JWT 토큰을 반환받음.
- **회원 정보 조회**: 인증된 사용자는 본인의 회원 정보를 확인할 수 있음.
- **회원 수정**: 사용자는 이메일, 닉네임 등의 정보를 수정할 수 있음.
- **회원 탈퇴**: 사용자는 탈퇴 신청을 할 수 있고, 관리자도 이를 처리할 수 있음.
- **중복 이메일 확인**: 회원 가입 시 이메일 중복 체크.
- **패스워드 변경**: 사용자가 비밀번호를 변경할 수 있도록 처리.

#### 기능 명세
- 사용자 인증을 위한 로그인/회원가입 API 구현.
- JWT를 사용한 인증과 토큰 관리.
- 비밀번호 해싱 및 보안 강화.
- 회원 정보 수정 및 삭제 처리.

---

### 2. 게시판 및 게시글 관리
- **게시판 CRUD**: 관리자는 게시판을 생성, 조회, 수정, 삭제할 수 있음.
- **게시판 목록 조회**: 사용자 또는 관리자는 전체 게시판 목록을 볼 수 있음.
- **게시글 작성**: 사용자는 게시글을 작성하고, 제목, 내용 등을 입력.
- **게시글 수정**: 게시글 작성자는 본인의 게시글을 수정할 수 있음.
- **게시글 삭제**: 게시글 작성자 및 관리자 모두 게시글을 삭제할 수 있음.
- **게시글 조회**: 게시글을 조회하고, 댓글을 달 수 있음.
- **댓글 관리**: 게시글에 댓글을 달 수 있고, 댓글을 삭제하거나 수정할 수 있음.
- **게시글 페이징**: 게시글 목록은 페이징 처리하여 효율적으로 조회.

#### 기능 명세
- 게시글 작성/수정/삭제/조회 API 구현.
- 댓글 작성/수정/삭제 API 구현.
- 게시글 조회 시 페이징 처리.
- 게시판 선택 시, 해당 게시판의 게시글만 필터링하여 조회.

---

### 3. 게시판 관리 (관리자)
- **게시판 목록 조회**: 관리자 화면에서 모든 게시판을 조회할 수 있음.
- **게시판 수정 및 삭제**: 관리자는 게시판을 수정하거나 삭제할 수 있음.
- **게시판 이름 변경**: 게시판의 이름을 변경할 수 있음.
- **게시판 삭제 시 데이터 처리**: 게시판 삭제 시 해당 게시판의 게시글, 댓글, 관련 데이터를 처리.

#### 기능 명세
- 게시판 CRUD API (관리자용).
- 게시판 삭제 시, 게시판에 속한 게시글과 댓글 삭제.

---

### 4. 회원 관리 (관리자)
- **회원 목록 조회**: 관리자 화면에서 모든 회원 정보를 볼 수 있음.
- **회원 삭제**: 관리자는 회원을 삭제할 수 있음.
- **회원 상세 정보 조회**: 회원의 상세 정보를 조회할 수 있음.

#### 기능 명세
- 관리자용 회원 관리 API 구현.
- 회원 삭제 시, 관련 게시글 및 댓글 처리.

---

### 5. 통계 및 실시간 데이터 (관리자)
- **실시간 트래픽**: 현재 시스템에서 발생하는 트래픽 데이터를 실시간으로 제공.
- **1일 접속자 수**: 하루 동안 접속한 고유 사용자 수를 제공.
- **게시글 및 사용자 활동 데이터**: 작성된 게시글 수, 댓글 수, 활동한 사용자 수 등을 제공.
- **서버 상태 및 에러 로그**: 서버 리소스(메모리, CPU 사용량)와 에러 로그를 제공.
- **사이트 방문자 통계**: 시간별/일별/주별 방문자 수.

#### 기능 명세
- 실시간 트래픽 API.
- 1일 접속자 수 API.
- 사용자 활동 데이터 API (게시글, 댓글 등).
- 서버 상태 및 에러 로그 API.

---

### 6. 알림 기능 (선택 사항)
- **이메일 알림**: 게시글, 댓글 작성 시 이메일로 알림을 보내줌.
- **푸시 알림**: 실시간으로 푸시 알림을 통해 새로운 게시글, 댓글 등을 알려줌.

#### 기능 명세
- 알림 서비스 구현 (이메일, 푸시).
- 알림 설정 API 제공 (사용자에게 알림을 보낼지 여부 설정).

---

### 7. 보안 및 권한 관리
- **JWT 기반 인증 및 권한 관리**: 모든 API는 JWT 토큰을 기반으로 인증하고, 관리자/일반 사용자의 권한을 구분.
- **API 보안 강화**: 비밀번호 암호화, 민감한 데이터 암호화 등 보안 기능 강화.
- **권한 설정**: 사용자가 일반 사용자, 관리자 등 권한에 따라 제한된 기능만 접근할 수 있도록 설정.
- **로그인 상태 유지**: 사용자 로그인 상태를 유지하여 인증된 상태에서만 접근할 수 있도록 설정.

---

### 8. 기타 기능()
- **백업 및 복구**: 데이터베이스 백업 및 복구 기능.
- **로그 관리**: 시스템 로그와 사용자의 행동 로그를 기록하여 나중에 분석할 수 있도록 제공.

---

### 기능 흐름도 및 관계도
1. **회원 관리**: 회원 가입 → 로그인 → 회원 정보 조회/수정/탈퇴 → 관리자 회원 관리 (회원 삭제)
2. **게시판 관리**: 게시판 CRUD (관리자) → 게시글 작성/수정/삭제 → 게시글 목록 조회 (페이징) → 댓글 작성/수정/삭제
3. **게시글 관리**: 게시글 조회 → 댓글 작성/수정/삭제 → 게시글 삭제
4. **통계 및 실시간 데이터**: 실시간 트래픽 → 1일 접속자 수 → 게시글/사용자 활동 데이터 → 서버 상태 및 에러 로그
5. **보안**: JWT 인증 → 권한 관리 → 사용자 인증 및 권한 부여
